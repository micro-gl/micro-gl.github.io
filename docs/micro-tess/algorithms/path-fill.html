Introduction

Path Fill Tessellates any collection of polygons of any type

Path is a modern interface for vector graphics, where you can define multiple paths with tools such as:

  1. Lines, Rectangles
  2. Bezier curves
  3. Elliptic arcs And then you can tessellate them automatically using:
  4. Fill Tessellation Planar Subdivision
  5. Stroke Tessellation Stroke Tessellation

It is best to see the example of usage below.

Notes:

  • Self contained and
    does not require stdlib
  • After Fill or Stroke Tessellation, internal buffers are cached, so re-tessellation will happen only if something is invalid. This is done to save energy.
  • Internal cache buffers can be drained using the
    drainBuffers()
     method
  • Supports Fill Rules -
    even-odd
     and
    non-zero
  • Numerically stable. Can tessellate with any number precision:
    Q
     (fixed point),
    float
     and
    double
  • Includes geometric optimizations
  • Allocator-aware so you can use it in any computer memory model (even on the stack)
  • 3 Configurable tessellation qualities
  • Debuggable messages flag with macro definition (this will require stdlib)
  • Debuggable optional approximate trapezes output
  • Optional boundary info output

Input

Input to the algorithm is

  • Fill rule enum
    microtess::fill_rule
  • Tess quality enum
    microtess::tess_quality
  • extra flags for debug info and boundary info
  • You can decide which template for internal container is used (such as std::vector)
  • You can decide which allocator is used
  • You can even use an array that is on the stack with
    static_array

Output

The algorithm outputs a

buffers
struct which holds containers for
vertices
,
indices
,
triangles type
,
boundary info
,
debug trapezes

Debugging

If you are seeing bugs with the tessellation, try the following

  • Set
    APPLY_MERGE
     template argument to
    false
    in
    tessellateFill
     method, this optimization might be producing problems.
  • Try increasing
    MAX_ITERATIONS
     template argument in
    tessellateFill
     method. Your algorithm might need more iterations if one of your polygons is huge. This argument is a fail safe way to exit an infinite loop if one shoudl occur.
  • Add
    #define MICROTESS_PLANAR_DEBUG_MESSAGES
     before all other includes. This should give messages and throw exceptions when the algorithm 'feels' things are wrong, so this can give you info.
  • Try increasing the precision of the
    number
    type of the vertices:
    • If using
      Q
      , try increasing precision bits. Q<8> -> Q<15>
    • If using
      float
      , then try
      double
       etc..

Example

taken from

examples/example_draw_path_fill.cpp

The example demonstrates usage of different linear containers (including the packaged non standard

dynamic_array
), also a usage with a static array, that manages all memory on the stack.

The example also demonstrates different number types such as fixed point

integer(
Q
 numbers), float and doubles. Of particular interest is
Q<4>
, a very low 4 bit precision, that can compute the tessellation, This is the first published tessellator, that can do that. Tessellation computation is only few lines of code.

I use the

microgl
project to render the tessellation.

30path_t<number> path_arc_animation() {
