Introduction

Planarize Division Tessellates any collection of polygons of any type

  • Supports Fill Rules -
    even-odd
     and
    non-zero
  • Numerically stable. Can tessellate with any number precision:
    Q
     (fixed point),
    float
     and
    double
  • Includes geometric optimizations
  • Allocator-aware so you can use it in any computer memory model
  • 3 Configurable tessellation qualities
  • Debuggable messages flag with macro definition (this will require stdlib)
  • Debuggable optional approximate trapezes output
  • Optional boundary info output
  • If randomization is incorporated at the algorithm edge-wise, this is the fastest algorithm on average with
    O(n*log(n))
    performance for any type of polygons and multi-polygons. Currently, the per edge randomization was disabled because it might make the implementation less stable, but I might re-add it with a flag. Also, partial randomization can be accomplished by randomizing the polygon.

Fill Rules

These can be chosen by the Enum

microtess::fill_rule

  • fill_rule::non_zero
     - only includes subdivisions, which have a non-zero winding number.
  • fill_rule::even_odd
     - only includes subdivisions, which have odd winding number.

Tessellation Quality

These can be chosen by the Enum

microtess::tess_quality

  • tess_quality::fine
     - The fastest algorithm, but may produce zero area triangles on the boundary because it fan triangulates the approximate trapezes. might be a problem if you are using SDF based AA
  • tess_quality::better
     - A bit slower and might be susceptible for other issues, but produces triangles out of each trapeze in a way similar to ear clipping, this fights zero area triangles on the boundary, so you can use SDF AA
  • tess_quality::prettier_with_extra_vertices
     - fast algorithm, that produces eye pleasing results, but uses around x2 memory for indices because it adds a center vertex in a trapeze and therefore adds two more triangles per trapeze on average
  • tess_quality::worst_visuals_but_fast_and_constant_memory
     (TBD) - the worst algorithm visually, as it will create 2 triangles per trapeze, BUT let's not forget each trapeze is an approximation, therefore cracks will show up in many cases. Also, this will have 2 triangles per trapeze so this is the most memory efficient

Input

Input to the algorithm is

  • Fill rule enum
    microtess::fill_rule
  • Tess quality enum
    microtess::tess_quality
  • Any iterable container which stores collections of collections of vertices, Each collection is a single polygon, and you can have as many as you want. In this example, I use my brewed container
    chunker
    , but you can use any standard one like a vector of vectors of vertices (
    std::vector<vector<vec2<number_type>>>
    ).
  • Output variables to be filled by the algorithm
  • Optional allocator for computation

Output

The algorithm output in the example below:

  • Will fill The
    output_triangles_type
     for you to know how to iterate the triangles (TRIANGLES/FAN/STRIP)
  • Will fill the user supplied
    output_vertices
     container with triangulation vertices.
  • Will fill the user supplied
    output_indices
     container with triangulation indices.
  • Will (optionally) fill the user supplied
    output_boundary_info
     container for you to know how to identify the boundary.
  • Will (optionally) fill the user supplied
    output_optional_approximate_trapezes
     container for you to roughly understand the trapezes.

Debugging

If you are seeing bugs with the tessellation, try the following

  • Set
    APPLY_MERGE
     template argument to
    false
    , this optimization might be producing problems.
  • Try increasing
    MAX_ITERATIONS
     template argument. Your algorithm might need more iterations if one of your polygons is huge. This argument is a fail safe way to exit an infinite loop if one shoudl occur.
  • Add
    #define MICROTESS_PLANAR_DEBUG_MESSAGES
     before all other includes. This should give messages and throw exceptions when the algorithm 'feels' things are wrong, so this can give you info.
  • Try increasing the precision of the
    number
    type of the vertices:
    • If using
      Q
      , try increasing precision bits. Q<8> -> Q<15>
    • If using
      float
      , then try
      double
       etc..

80    auto render_polygon = [&](const chunker_t<number> & pieces) {
81        using index = unsigned int;
82
83        // Algorithm START
84        // output tess vertices
85        container<vertex2<number>> output_vertices;
86        // output tess indices
87        container<index> output_indices;
88        // output optional approximate trapezes
89        container<vertex2<number>> output_optional_approximate_trapezes;
90        // output optional boundary info
91        container<microtess::triangles::boundary_info> output_boundary;
92        // output triangles indices type
93        microtess::triangles::indices output_triangles_type;
94        // optional computation allocator
95        microtess::std_rebind_allocator<> allocator;
96        // define algorithm
97        using psd = microtess::planarize_division<
98                            number,
99                            decltype(output_vertices),
100                            decltype(output_indices),
101                            decltype(output_boundary)>;
102                            decltype(allocator)>;
103        // compute algorithm
104        psd::template compute<decltype(pieces)>(pieces,
105                        microtess::fill_rule::even_odd,
106                        microtess::tess_quality::better,
107                        output_vertices, 
108                        output_triangles_type, 
109                        output_indices,
110                        &output_boundary, 
111                        &output_optional_approximate_trapezes,
112                        allocator);
113        // Algorithm END
114
115        canvas.clear({255, 255, 255, 255});
116        canvas.drawTriangles<blendmode::Normal, porterduff::None<>, false>(
117                color_red,
118                matrix_3x3<number>::identity(),
119                output_vertices.data(),
120                (vertex2<number> *)nullptr,
121                output_indices.data(),
122                output_boundary.data(),
123                output_indices.size(),
124                output_triangles_type,
125                255);
126
127        canvas.drawTrianglesWireframe({0,0,0,255},
128                            matrix_3x3<number>::identity(),
129                            output_vertices.data(),
130                            output_indices.data(),
131                            output_indices.size(),
132                            output_triangles_type,
133                            40);
134
135        for (index ix = 0; ix < output_optional_approximate_trapezes.size(); ix+=4)
136            canvas.drawWuLinePath({0, 0, 0, 255},
137                                  &output_optional_approximate_trapezes[ix], 4, true);
138    };
139
140    auto render = [&](void*, void*, void*) -> void {
141        static auto polygons = poly_inter_star_2<number>();
142        render_polygon(polygons);
143    };
144
145    example_run(&canvas, render);
146}
