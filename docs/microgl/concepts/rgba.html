Introduction

Currently, microgl works with the following objects to define color

RGBA info

  • Defined at
    <microgl/color.h>
    , at the
    microgl
    namespace

The

rgba_t
template is roughly defined as 

template<uint8_t R, uint8_t G, uint8_t B, uint8_t A>
struct rgba_t {
    static constexpr uint8_t r = R;
    static constexpr uint8_t g = G;
    static constexpr uint8_t b = B;
    static constexpr uint8_t a = A;
};

This structure holds info about the bit depth and is used by pixel encoder, samplers and canvas to know at compile time the bit depth of each channel and perform correct and fast math operations.

Usage is as simple as

#include <microgl/color.h>

using RGB_5650 = microgl::rgba_t<5,6,5,0>;

auto r_bits = RGB_5650::r;
auto g_bits = RGB_5650::g;
auto b_bits = RGB_5650::b;

Color

  • defined at
    <microgl/color.h>
    , at the
    microgl
    namespace

Color is a simple struct that holds 4 channels of size 1 byte each. the color object does not hold the bit depth of the colors. It assumes, that it is known by the user.

struct color_t {
    color_t(channel_t $r=0, channel_t $g=0, channel_t $b=0, channel_t $a=255)
    color_t(const color_t &val)
    uint8_t r, g, b, a;
};

color bit depth generally ranges from

0
up to
(2^bits)-1
, For example, if you have:

  • 8 bit channel, then the values it take range from
    0 to 255
  • 7 bit channel, then the values it take range from
    0 to 127
  • 6 bit channel, then the values it take range from
    0 to 63
  • 5 bit channel, then the values it take range from
    0 to 31
  • 4 bit channel, then the values it take range from
    0 to 15
  • 3 bit channel, then the values it take range from
    0 to 7
  • 2 bit channel, then the values it take range from
    0 to 3
  • 1 bit channel, then the values it take range from
    0 to 1
  • 0 bit channel, then the values it take range from
    0 to 0

Usually, at most graphic systems, colors are stored as intensities ranging from

0..1
per channel, afterwards, when storing the intensity, it will be quantized or multiplied by the maximum value of the bit depth, for example
rgb(0.5, 1, 1) => rgb(128, 255, 255)
, BUT, in
microgl
, colors are not processed as intensities at the the pipeline, because we feel it may hurt performance to add divisions/multiplications per read/write operations. Therefore, atm, colors are not stored as intensities, but as actual quantized values. This might change in the future, while we explore performant ways to do it and also extend the type of the channels to contain more bytes if needed.

Intensity

microgl also defines intensity template defined at

<microgl/color.h>


intensity<float> in{0.3f, 0.2f, 1.0f}
intensity<Q<15>> in{ Q<15>(1)/3, Q<15>(1)/5, Q<15>(1)}

We are exploring ways of making intensities more performant so we can use them more in the pipeline.

Other methods in
color.h

Here are some highlights

convert_color

/**
* convert a color of one depth space into another 
* 
* @tparam rgba_from the rgba_t info of the input color 
* @tparam rgba_to the rgba_t info of the output color
* @param input input color
* @param output output color
*/
template<typename rgba_from, typename rgba_to>
void convert_color(const color_t &input, color_t &output)

convert_intensity_to_color

/**
* convert an intensity to color, example :
*
* Example: intensity<float>(0.5, 1.0, 1.0f) -> color_t{128, 255, 255}
*
* @tparam number the underlying type of the intensity
* @tparam rgba the rgba_t info we desire to quantize
* @param input the input intensity
* @param output output color
*/
template <typename number, typename rgba>
void convert_intensity_to_color(const intensity<number> &input, color_t &output)

convert_color_to_intensity

/**
* convert a color into intensity object
*
* Example: color_t{128, 255, 255} -> intensity<float>(0.5, 1.0, 1.0f)
*
* @tparam number number the underlying type of the intensity
* @tparam rgba the rgba_t info of the color
* @param input input color
* @param output output intensity
*/
template <typename number, typename rgba>
void convert_color_to_intensity(const color_t &input, intensity<number> &output)
